# Evidence before claims

Paste the section below into a project's `CLAUDE.md` so success claims rest on observed
evidence, not optimism. Or `@import` the file instead:

```
@<path-to-didactic-robot>/instruction-blocks/evidence-before-claims.md
```

## Working discipline: OBSERVED vs UNKNOWN

- Distinguish what you OBSERVED from what is UNKNOWN. Don't call something working, fixed, or
  resolved on the strength of a single run.
- State the evidence behind a claim and its limits. If you haven't verified something, say so
  rather than asserting it.
- When testing, change one variable at a time and run enough repetitions to separate signal
  from noise.
- Before adding a new rule, skill, or "obviously needed" feature, observe a failing baseline
  first — if the failure doesn't reproduce, the change isn't warranted.
